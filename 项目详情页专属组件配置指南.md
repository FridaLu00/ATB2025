# 项目详情页专属组件配置指南

## 📋 目录结构

```
src/app/project/[id]/
├── page.tsx                    # 主布局文件（路由入口）
└── content/                    # 内容组件目录
    ├── ProjectContent.tsx      # 通用内容组件（所有项目默认使用）
    ├── Project1Content.tsx     # 项目1专属组件
    ├── Project2Content.tsx     # 项目2专属组件（待创建）
    └── ...
```

---

## 🎯 工作原理

### 数据流

```
用户访问 /project/5
    ↓
page.tsx 获取 ID = 5
    ↓
从 @/lib/projects.ts 查找 id=5 的项目数据
    ↓
根据 ID 选择对应的内容组件
    ↓
组件使用 project 数据渲染内容
```

### 组件选择逻辑

在 `page.tsx` 中定义了组件映射：

```tsx
const contentComponents: Record<number, React.ComponentType<{ project: Project }>> = {
  1: Project1Content,
  // 2: Project2Content,
  // 3: Project3Content,
  // ...
};

const ContentComponent = contentComponents[projectId] || ProjectContent;
```

- 如果项目ID在映射中，使用专属组件
- 如果项目ID不在映射中，使用通用组件 `ProjectContent`

### 步骤 1：创建专属内容组件

复制 `Project1Content.tsx` 作为模板：

打开新创建的文件，修改以下内容：

#### 2.1 修改组件名称

```tsx

// 修改后

```

#### 2.2 自定义布局结构

根据项目需求修改 JSX 结构，例如：

```tsx
export default function Project2Content({ project }: ProjectContentProps) {
```

## ⚠️ 注意事项

### 1. 底部预留空间

所有内容组件必须保留底部预留空间，避免被引导窗口遮挡：

```tsx
<div className="pb-40">
  {/* 内容 */}
</div>
```

### 2. TypeScript 类型

组件必须正确接收 `project` 参数：

```tsx
interface ProjectContentProps {
  project: Project;
}

export default function ProjectXContent({ project }: ProjectContentProps) {
  // ...
}
```

### 3. 数据来源

所有项目数据来自 `src/lib/projects.ts`，不要在组件中硬编码数据。

### 4. 响应式设计

确保布局在移动端和桌面端都能正常显示。

## 🎨 常见布局模式

### 模式 1：通用模板（默认）

使用 `ProjectContent.tsx`，适合大多数项目。

### 模式 2：专属布局

为特定项目创建专属组件，适合需要特殊设计的项目。

### 模式 3：混合模式

部分项目使用通用模板，部分项目使用专属布局。

## 📞 常见问题

### Q: 如何知道当前使用的是哪个组件？

A: 在组件中添加 `console.log`：

```tsx
export default function Project1Content({ project }: ProjectContentProps) {
  console.log("使用 Project1Content");
  // ...
}
```

### Q: 可以删除 `Project1Content.tsx` 吗？

A: 可以，但需要先从 `page.tsx` 的导入和映射中移除。

### Q: 如何让所有项目都使用通用模板？

A: 删除 `page.tsx` 中的 `contentComponents` 映射，直接使用：

```tsx
<ProjectContent project={project} />
```
